AI ENABLEMENT · TASK 1

HireIQ

AI-Powered Candidate Intelligence

An agentic pipeline that parses job descriptions and resumes, scores candidates across 5 dimensions using a structured rubric, and produces transparent, auditable shortlist reports.

THE PROBLEM

Manual Screening is Broken

Fatigue

HR teams screen 200+ applications per role. Quality drops after the first 50.

Inconsistency

Different reviewers apply different criteria. No standardised rubric.

Unconscious Bias

Name, university, gender can influence decisions without awareness.

Time Cost

Average time-to-shortlist: 5–7 days. With AI: under 2 minutes.

THE SOLUTION

Agentic Evaluation Pipeline

  • Upload JD + resumes → agent parses both into structured Pydantic models
  • 5-dimension scoring rubric with explicit numeric bands (0-3, 4-6, 7-10)
  • Transparent justifications for every score — no black-box decisions
  • Human-in-the-loop: HR can override any score with a logged reason
  • Interactive HTML report with colour-coded score bars and recommendations
ARCHITECTURE

Pipeline Flow

JD Upload → JD Parser Agent → JobRequirements (Pydantic) ↓ Resumes → Profile Parser → CandidateProfile (Pydantic) ↓ Scorer Agent → CandidateScore (5 dimensions) ↓ Ranker → ShortlistReport (sorted) ↓ Report Generator → HTML + JSON output ↓ HR Override Panel → Audit-logged adjustments

LEARNINGS

Key Takeaways

  • Structured output (Pydantic) eliminates 90% of LLM parsing headaches
  • Prompt iteration is critical — v5 was dramatically better than v1
  • SQLite caching saved ~$15 in API costs during development
  • Bias mitigation is a design choice, not an afterthought
  • LangSmith tracing is invaluable for debugging scoring inconsistencies
  • Rate limiting requires built-in pauses for free-tier API keys
